# GameStack ‚Äî Setup Checklist

Complete list of every placeholder value in the repo, what it means, and
exactly where to find it. Work top to bottom. Items marked **Required** will
prevent the stack from starting if left as-is. Items marked **Display only**
are cosmetic ‚Äî the stack runs fine without them, but they'll show placeholder
text in the portal UI.

---

## 1. `.env` ‚Äî **Required**

Create first. Everything else depends on it.

```bash
cp .env.example .env
```

| Line | Field | Placeholder | What to set |
|------|-------|-------------|-------------|
| 7 | `TZ` | `America/Los_Angeles` | Your timezone ‚Äî [full list](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones) |
| 12 | `DATA_DIR` | `~/Gamestack` | Absolute path to your Gamestack dir. Use `~/Gamestack` if cloned to home. |
| 15 | `AMP_USERNAME` | `your_amp_username` | AMP login username |
| 16 | `AMP_PASSWORD` | `your_amp_password` | AMP login password |
| 17 | `AMP_LICENCE` | `your_amp_licence_key` | AMP licence key from the AMP portal |
| 21 | `AMP_MAC` | `02:42:xx:xx:xx:xx` | Fixed MAC address for AMP container ‚Äî prevents licence deactivation on restart. Generate: `printf '02:42:%02x:%02x:%02x:%02x\n' $((RANDOM%256)) $((RANDOM%256)) $((RANDOM%256)) $((RANDOM%256))` |

`GAME_PORT_1` (9876) and `GAME_PORT_2` (9877) are defaults ‚Äî change to
match the ports your game server actually uses.

## 2. `wolf-config/cfg/config.toml` ‚Äî Required on migration, auto on fresh install

On a **fresh install**: `setup.sh` copies `config.toml.example` to
`wolf-config/cfg/config.toml` automatically. Wolf will auto-populate `uuid`,
`app_state_folder`, and `client_cert` when it first starts and clients pair.
No manual changes needed.

On a **migration** from an existing install: replace the example values with
your real ones from the old machine.

| Line | Field | Placeholder | Notes |
|------|-------|-------------|-------|
| 3 | `uuid` | `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` | Auto-generated by Wolf on first start |
| 164, 179, 194 | `app_state_folder` | `0000000000000000000` | Auto-populated when Moonlight clients pair |
| 166, 181, 196 | `client_cert` | `<auto-populated...>` | Auto-written by Wolf when clients pair |

---

## 3. `wolf-config/cfg/cert.pem` and `key.pem` ‚Äî Auto on fresh install

Wolf generates these TLS files automatically on first start. The `.example`
files in the repo are placeholders only ‚Äî **do not** create these manually.
`setup.sh` will not overwrite them if they already exist.

## 4. `ampdata/instances.json` ‚Äî Auto on fresh install

`setup.sh` copies `instances.json.example` to `ampdata/instances.json` if
none exists. AMP then manages this file itself. The UUIDs in the example are
zeroed out ‚Äî AMP regenerates them on first start.

| Lines | Field | Notes |
|-------|-------|-------|
| 30‚Äì32, 127‚Äì129 | `InstanceID`, `TargetID`, `CreatedBy` | All `00000000-...` ‚Äî AMP auto-generates |
| 67, 164 | `MetricsPublishingHMAC` | Auto-generated by AMP |

## 5. `portal/html/index.html` ‚Äî Display only

The portal auto-detects your host IP from `window.location.hostname` for all
functional features (health checks, AMP iframe, stream launcher). The items
below are **display strings only** ‚Äî update them so the portal shows your
actual network info instead of placeholders.

Quickest approach ‚Äî run these from `~/Gamestack` after filling in your values:

```bash
HOST_IP=192.168.x.x           # Your host's LAN IP
PUBLIC_IP=1.2.3.4             # Your WAN/public IP
DOMAIN=yourdomain.com         # Your public domain
GAME_SUBDOMAIN=game           # Subdomain for game server (game.yourdomain.com)
USERNAME=youruser             # Your username
NETWORK=YourNetwork           # Your network or VPN name
HOSTNAME=your-hostname        # Your machine's hostname
ROUTER_IP=192.168.x.1         # Your router's LAN IP
UPSTREAM_ROUTER_IP=192.168.x.x # Upstream router (if double-NAT)
MODEM_IP=192.168.x.x          # ISP modem/gateway IP

sed -i "s/your-username \/ YourNetwork/${USERNAME} \/ ${NETWORK}/g"   portal/html/index.html
sed -i "s/your-hostname/${HOSTNAME}/g"                                 portal/html/index.html
sed -i "s/YOUR_HOST_IP/${HOST_IP}/g"                                   portal/html/index.html
sed -i "s/YOUR_PUBLIC_IP/${PUBLIC_IP}/g"                               portal/html/index.html
sed -i "s/YOUR_ROUTER_IP/${ROUTER_IP}/g"                               portal/html/index.html
sed -i "s/YOUR_UPSTREAM_ROUTER_IP/${UPSTREAM_ROUTER_IP}/g"             portal/html/index.html
sed -i "s/YOUR_MODEM_IP/${MODEM_IP}/g"                                 portal/html/index.html
sed -i "s/game\.yourdomain\.com/${GAME_SUBDOMAIN}.${DOMAIN}/g"        portal/html/index.html
sed -i "s/yourdomain\.com/${DOMAIN}/g"                                 portal/html/index.html
sed -i "s/YourNetwork/${NETWORK}/g"                                    portal/html/index.html
```

Or edit manually ‚Äî exact locations:

## 6. `build.js` ‚Äî Personal ops notebook, not a template

`build.js` contains your personal hardware inventory, network topology, and
session notes for generating `output/GameStack.html`. It already has generic
placeholders applied. Edit it freely to document your own setup ‚Äî it is not
used by the running stack, only by `node build.js` to regenerate docs.

---

## Summary ‚Äî minimum required to start the stack

| Step | Required |
|------|----------|
| Create `.env` from `.env.example`, fill in all fields | **Yes ‚Äî stack will not start** |
| Copy `config.toml.example` ‚Üí `wolf-config/cfg/config.toml` | Auto ‚Äî `setup.sh` handles this |
| Copy `instances.json.example` ‚Üí `ampdata/instances.json` | Auto ‚Äî `setup.sh` handles this |
| Update portal display strings (section 5) | Optional ‚Äî portal works, just shows placeholders |
| Personalise `build.js` with your hardware/network | Optional ‚Äî only affects generated docs |

Run `bash setup.sh` after creating `.env` ‚Äî it handles the rest.
